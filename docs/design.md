# FloatSwitch 設計仕様書

最終更新: 2026-02-24

---

## 1. 概要

### 1.1 背景・課題

macOS 標準の `Command + Tab` によるウィンドウ切り替えには以下の問題がある。

- キーボードのみで完結しない（Enter キーで切り替えられない）
- 最小化ウィンドウが視認しづらく、切り替えに手間がかかる
- 操作のたびにマウス移動が発生し、作業フローが途切れる

### 1.2 コンセプト

**「常に最前面のミニマルなフローティングタブバー」**

`Command + Tab` の視覚的な代替として、画面端に常駐するフローティングバーを提供する。
アイコンクリックまたはホットキーで瞬時にウィンドウを切り替えられ、マウス移動を最小限に抑える。

## 3. UI / UX 仕様

### 3.1 フローティングバー

- 常に最前面に表示（他のウィンドウに隠れない）
- **デフォルト位置**: 画面右下
- **移動**: ドラッグで任意の位置に移動可能

### 3.2 レイアウト

2段構成でアプリとフォルダを分離する。

```
┌─────────────────────────────┐
│  [Icon] AppA  [Icon] AppB   │  ← 上段: アプリ
├─────────────────────────────┤
│  [Icon] FolderA  [Icon] FolderB │  ← 下段: Finder フォルダ
└─────────────────────────────┘
```

- 各アイテムはアプリアイコン + アプリ名を**常時表示**（ホバー不要）

### 3.3 非アクティブ時の挙動

- カーソルがバー外に出ると**半透明**になる
- バーのサイズや位置は変化しない（折りたたみなし）

## 4. 機能仕様

### 4.1 表示対象

| 種別 | 対象 |
|------|------|
| アプリ（上段） | 現在起動中の全アプリ（最小化含む） |
| フォルダ（下段） | Finder で開いているウィンドウ（最小化含む） |

### 4.2 ウィンドウ切り替え

- アイコンをクリックするとそのウィンドウをフォーカス・最前面に呼び出す
- 最小化されているウィンドウも復元してフォーカスする

### 4.3 複数ウィンドウの扱い

同一アプリが複数ウィンドウを持つ場合：

- バー上のアイコンは1つにまとめる
- クリック時にサブメニューを表示し、ウィンドウを選択して切り替える

### 4.4 ホットキー

- `Command + Tab` の代替として機能することを目的とする
- **デフォルト**: `Cmd + 数字（1〜9）` で上段アプリを順番に切り替え
- ホットキーはユーザーがカスタマイズ可能
- ホットキーカスタマイズ機能はアプリ内の設定画面から行う（詳細は未確定）

## 6. 非機能要件

- 作業領域を極力妨げないこと（省スペース設計）
- 常時起動を前提とした低リソース消費
